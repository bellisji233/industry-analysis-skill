# 行业分析 Skill

帮助用户**看懂一个产业 + 找到产业内的投资机会**。最终交付一份自包含的 HTML 可视化研究报告。

## 核心原则

1. **先回显、再开干**：阶段 0 先输出对行业边界的理解，让用户确认后再做全量分析，避免方向跑偏。
2. **结构化产出，不给空表**：直接给出打分、判断和结论，而不是丢一个框架让用户自己填。
3. **联网取数、标注来源**：市场规模、CAGR、毛利率、市占率、标的财务等关键数字用 `WebSearch` 拉取最新数据，正文标注来源与时间。结构性判断（产业链结构、三高逻辑）可用既有知识。
4. **A 股为主视角**：标的、国产化率、政策催化默认以中国 A 股视角展开，兼顾港股；用户另有指定则从其要求。
5. **投资合规**：做"逻辑映射 + 跟踪框架"，不做"买入/卖出推荐"。每份报告必须包含风险与证伪条件。报告底部附免责声明。

## 工作流（6 阶段）

### 阶段 0 · 范围界定（必须先回显确认）
理解并回显以下要点，给出你的默认假设，请用户确认或修正后再继续：
- **行业定义与边界**：含/不含哪些环节（举例说明易混淆的边界）
- **分析视角**：产业理解 / 投资择股（默认两者，偏重投资）
- **市场重心**：默认 A 股为主、兼顾港股
- **核心环节是否在境外（必判）**：若该产业的价值高地/关键玩家在海外（如 AI 算力、半导体设备、高端材料的英伟达/台积电/ASML 等），**产业链图谱必须纳入海外龙头做定位**，标的映射再回落到 A 股受益/替代标的。不要因"A 股视角"而漏掉真正拿走利润的环节。
- **周期位置**：你初判该产业处于成长期/成熟期/出清期
- **下钻深度**：默认下钻 1–2 层细分

> 若用户已在提问中明确了边界与重心，可一句话回显默认假设并直接进入阶段 1，不必反复确认。

### 阶段 1 · 产业链图谱（上游 → 中游 → 下游）
- 拆出**上游**（原材料/核心零部件/关键设备/技术IP）、**中游**（制造/加工/集成/封装）、**下游**（应用场景/渠道/终端需求）
- 每个环节标注：环节名称、典型玩家（A股优先）、**价值量占比**、**供给格局/国产化率**
- 联网核验：市场规模、各环节价值量分布、国产化率
- 产出：横向产业链 SVG 图谱，按价值量做热力上色（见 assets/report-template.html 组件）

### 阶段 2 · 高价值方向识别（"三高"打分）
对每个环节按三维打分（1–5 分），打分规则见 `references/methodology.md`：
- **高增长**：行业 CAGR、需求驱动力、渗透率/S 曲线位置、政策催化
- **高利润**：毛利率/净利率、ROIC、微笑曲线位置、价值捕获能力
- **高壁垒**：技术/专利、资本开支、认证周期、客户切换成本、规模/品牌
- 叠加投资维度：议价能力、价值量占比、**国产替代空间**
- **财务交叉校验（必做）**：每个维度联网取客观财务锚（增长=营收CAGR/增速、利润=毛利率/净利率/ROIC、壁垒=CR集中度/市占/研发费用率），与主观分对照，标注校验状态 ✅一致 / ⚠️偏乐观 / 🔵偏保守 / ❓数据缺失。**偏乐观必须写明前瞻理由，否则下修到数据区间**。**两步走：先裸打分→再独立审计→改分留痕（`4→3`）；"校验状态"与"趋势备注"分列，别把"毛利下滑"等趋势误标成 ⚠️偏乐观**。细则见 `references/methodology.md` 第六节（6.0 两步走、6.2 状态 vs 趋势）。
- **价值迁移判断（必做）**：三高是当下快照，本步补时间维度——判断每个环节价值量/利润在未来 1–3 年是 **↑迁入 / ↓迁出 / →稳定**，并给一句驱动（一体化 vs 解耦、技术路线变迁、国产替代后利润再分配、上下游议价权转移）。**关键规则：当前高价值 + ↑迁入 = 最优；当前高价值 + ↓迁出 = 警惕已见顶（往往已被 price in，对应估值的负预期差）；当前中低价值 + ↑迁入 = 潜在预期差机会**。细则见 `references/methodology.md` 第九节。
- 产出：环节 × 三高 **热力矩阵**（含"价值迁移 ↑/↓/→"列）+ **多环节对比雷达图**（叠加 2–5 个高价值环节比形状差异，单环节不画——无增量信息）+ **财务校验表**（逐环节列实测锚值、来源时间、校验状态 + 独立的趋势备注列），自动高亮"高价值象限"，并标出"高价值且迁入"的最优象限

### 阶段 3 · 价值方向下探（细分赛道）
对筛出的高价值环节继续下钻：
- 拆到具体**细分技术路线 / 卡位点**（如 储能→电池→电芯→大圆柱/方形/软包）
- 每条路线：竞争格局、技术成熟度、确定性 vs 弹性
- 产出：高价值方向 → 细分方向 → 卡位点 的**树状下钻图**

### 阶段 4 · 产业标的梳理
按角色分类（方法见 `references/target-framework.md`），**做逻辑映射，不做推荐**：
- **龙头/确定性**、**弹性标的**、**隐形冠军**
- 每个标的：所处环节、主营结构、市占率、核心逻辑、催化剂、估值锚
- 联网核验：标的最新营收结构、市占率、估值
- 产出：标的—产业链定位**映射表**

### 阶段 5 · 投资机会综合 + 估值赔率 + 空头视角 + 风险/证伪
- 自上而下提炼 1–3 条**投资主线**
- **估值与赔率（必做）**：对核心环节/标的算 PEG、历史估值分位、隐含预期（预期差），落到"价值 × 估值"四象限，给赔率结论。**好行业≠好价格**——高价值但高估值要标注"赔率不足"。细则见 `references/methodology.md` 第七节。
  - ⚠️ **防编造硬约束**：PEG、历史估值分位这类需要"未来净利预测/历史 PE 序列"的数字最易滋生幻觉。**优先用 `assets/fetch_valuation.py` 取真实历史分位（methodology 7.2.1，脚本缺依赖不自动装、征得同意）；脚本取不到才标 ❓ 降级为定性区间（高/中/低），禁止凭空给出具体数值**（细则见 methodology 7.2.2）。
- **空头视角（必做，反共识对冲）**：本 skill 天然容易跟随共识叙事，故每份报告必须独立产出一个 **Bear Case 卡片**——这条主线最可能错在哪、空头最强的三条论据、什么数据出现即证伪。见 methodology 第八节。
- 每条主线写明**驱动假设**与**证伪条件**（出现什么数据就该退出）
- 产出：投资主线卡片 + **价值×估值四象限图 + 估值分位表** + **Bear Case 卡片** + 周期位置、风险点、需持续跟踪的关键指标清单

### 阶段 6 · 尽调追问清单 + 产出 HTML 报告
- **尽调/待验证清单（必做，本 skill 最高价值、最低幻觉风险的输出）**：报告末尾列出——① 哪些关键数字置信度低、需去年报/Wind/一手渠道核实；② 哪些假设一旦证伪整条结论就翻；③ 建议用户下一步该问的 3–5 个问题。把"帮你提出该问的问题"落成可执行清单。
- **数据置信度总评（必做）**：报告头部标一行整体置信度——本报告核心数据 X 项已联网核验 / Y 项为模型定性判断 / Z 项数据缺失，让用户一眼知道能信几分。
- **名词解释表（必做，新手友好）**：报告末尾（免责声明前）附一张"术语 → 一句话解释"表，覆盖本报告实际用到的金融/产业黑话（如 Beta/Alpha、卡位点、卖铲子、price in、预期差、PEG、历史分位、微笑曲线、CR3/CR5、渗透率/S曲线、国产替代/卡脖子等），照 `assets/report-template.html` 的固定表按需增删，让非专业用户也读得懂。
- 基于 `assets/report-template.html` 的结构与样式，生成自包含 HTML（内联 CSS/SVG，离线可打开、可截图）。各模块对应阶段 1–5 的产出。文件名形如 `<行业名>行业分析报告.html`，写到当前工作目录或用户指定路径，并告知路径。

- `examples/网络安全行业分析报告.html` — 软件/SaaS 类完整示例（PS/订阅/续费视角，含财务校验表、置信度总评、Bear Case、尽调清单）；分析非制造类行业时优先参考此例

## 取数建议
- 优先检索：行业市场规模与 CAGR、各环节价值量占比、国产化率、龙头市占率、标的毛利率/营收结构/估值。
- **三高校验专项取数**：每个高价值环节取龙头近 2–3 年营收增速、毛利率/净利率/ROIC、CR3/CR5 集中度、研发费用率——用于交叉校验主观打分。预测值与历史实际值分开标注。
- 检索词组合"行业名 + 市场规模/产业链/价值量/国产化率/竞争格局 + 年份"。
- 数字旁标注来源机构与时间；多源冲突时取较权威者并注明区间。
- **估值分位优先用脚本**：`python assets/fetch_valuation.py <代码...> --md` 直接取真实历史 PE/PB/PS 分位（akshare/东财）；脚本取不到再回退定性判断，别猜数（细则见 methodology 7.2.1）。
