# 分析方法论：三高打分 · 微笑曲线 · 价值捕获

本文件定义"高价值方向识别"的可操作标准。打分是为了**排序与高亮**，不是精确科学——重在拉开差距、给出可解释的理由。

## 一、"三高"打分（每维 1–5 分）

对产业链每个环节打三个分，并给一句话理由。

### 高增长（Growth）
| 分数 | 判据 |
|---|---|
| 5 | 行业/环节 CAGR ≥ 30%，渗透率处于 S 曲线陡升段，强政策+需求双驱动 |
| 4 | CAGR 15–30%，渗透率加速，明确催化 |
| 3 | CAGR 8–15%，稳健成长 |
| 2 | CAGR 3–8%，接近 GDP 增速 |
| 1 | 零增长或下滑、产能出清中 |

观察要点：需求驱动力（替代/渗透/扩容/政策）、渗透率当前位置、是否有第二增长曲线。

### 高利润（Profit）
| 分数 | 判据 |
|---|---|
| 5 | 毛利率 ≥ 50% 或 ROIC ≥ 20%，强价值捕获，提价能力强 |
| 4 | 毛利率 35–50%，盈利稳定且能扩张 |
| 3 | 毛利率 20–35%，行业平均 |
| 2 | 毛利率 10–20%，价格战压力 |
| 1 | 毛利率 < 10% 或亏损，纯加工/代工 |

观察要点：在**微笑曲线**中的位置、议价能力（对上游/下游）、价值捕获能力。

### 高壁垒（Barrier）
| 分数 | 判据 |
|---|---|
| 5 | 技术/专利+认证+客户绑定多重壁垒，新进入者 ≥ 3 年难追，CR3 高 |
| 4 | 单一强壁垒（核心专利/长认证周期/高资本开支） |
| 3 | 中等壁垒，规模或品牌护城河 |
| 2 | 低壁垒，主要靠成本/产能 |
| 1 | 几乎无壁垒，充分竞争 |

壁垒类型清单：技术/专利、资本开支强度、认证/资质周期、客户切换成本、规模经济、品牌、网络效应、数据。

## 二、投资叠加维度（不计入三高，但影响高亮）

- **价值量占比**：该环节在整链中拿走多少钱（决定弹性大小）
- **议价能力**：相对上下游的话语权（决定利润可持续性）
- **国产替代空间**：当前国产化率低 + 政策推动 = 强催化（A 股核心阿尔法来源）
- **供给格局**：CR3/CR5 集中度，越集中越易出龙头

## 三、微笑曲线（判断利润分布）

典型制造业产业链，利润呈"两端高、中间低"的微笑形：
- **左端（高）**：研发/核心材料/专利/设备 —— 技术壁垒高
- **中间（低）**：组装/制造/代工 —— 同质化、价格战
- **右端（高）**：品牌/渠道/服务/数据 —— 客户与场景壁垒

判断一个环节是否高价值：先定位它在微笑曲线的位置，再看议价能力是否支撑利润。
注意例外：强周期上游（如稀缺资源）在景气期利润极高但不可持续，要标注周期位置。

## 四、高价值象限的判定

综合三高总分（满分 15）+ 价值量占比 + 国产替代空间：
- **总分 ≥ 12 且 价值量占比高** → 核心高价值环节（报告重点高亮）
- **总分 ≥ 12 但 价值量占比低** → 隐形冠军型机会（弹性来自高 ROE/绝对龙头）
- **国产化率 < 30% 且 总分 ≥ 10** → 国产替代主线候选

## 五、周期位置标注（避免"高景气陷阱"）

任何"高利润"结论都要叠加周期判断：
- 产能扩张期 / 供需紧平衡 / 产能过剩出清期
- 在出清期，低估值 + 格局优化的环节反而是机会
- 在景气顶部，高利润不可持续，需提示均值回归风险

## 六、财务数据交叉校验（每次打分必做）

三高分是结构化的主观判断；为避免"拍脑袋"，**每个维度必须联网取至少一个客观财务锚与主观分对照**。
这一步是本 skill 区别于"裸模型随口打分"的核心：主观分要么被数据印证，要么被强制解释或下修。

### 6.0 两步走：先裸打分，再审计回改（防自我印证）

**顺序是硬要求**——若边找数据边打分，校验会退化成"自我印证"，失去审计意义。因此：

1. **第一步 · 裸打分**：先只凭产业结构判断打出三高分（暂不看具体财务数字），落成初稿分。
2. **第二步 · 独立审计**：再单独拉财务锚（6.1）与初稿分对照，按 6.3 处理背离。
3. **分数留痕（关键）**：审计若改动了分数，**必须在报告里留下修改痕迹**，写成 `4→3`（形式：初稿分→终稿分，附一句理由，如"利润 4→3，因龙头毛利实测 34%"）。没有留痕 = 没真正审计。分数未变则标 ✅ 说明数据印证。

### 6.1 维度 → 客观财务锚 映射

| 三高维度 | 必取的客观锚（联网） | 量化对照（与 1–5 分对应）|
|---|---|---|
| **高增长** | 代表性公司近 2–3 年营收 CAGR、行业 CAGR、渗透率 | 对照"一、高增长"区间表：≥30%→5，15–30%→4，8–15%→3，3–8%→2，<3%→1 |
| **高利润** | 代表性公司毛利率、净利率、ROIC/ROE | 对照"一、高利润"区间表：毛利≥50%→5，35–50%→4，20–35%→3，10–20%→2，<10%→1 |
| **高壁垒** | CR3/CR5 集中度、龙头市占率、研发费用率、毛利率的稳定性 | 高集中(CR3≥60%)+高且稳毛利+高研发率→4–5；分散+低毛利→1–2 |

> 取"代表性公司"时优先选该环节 A 股龙头；龙头数据缺失时取行业均值或可比公司，并注明。

### 6.2 校验状态（报告中需逐环节标注）

**校验状态 = "分数 vs 数据的对齐度"，只回答一件事：主观分有没有偏离客观锚。**
- ✅ **一致**：主观分落在数据对应区间（±1 分内）。
- ⚠️ **偏乐观**：主观分高于数据 ≥2 分（典型：给了高增长分，但龙头营收增速并不高）。
- 🔵 **偏保守**：主观分低于数据 ≥2 分（数据比印象更好，可能是被低估的机会）。
- ❓ **数据缺失**：取不到可靠财务锚，置信度下调，结论需打折。

> ⚠️ **不要把"趋势警示"误标成"背离"**（本 skill 常见错误）：如毛利率 34% 且利润打 3 分，按区间表本就是 ✅一致；即便毛利率"同比下滑"，那是**趋势/周期信号**，属于第 6.3.4 的备注，**不是分数与数据背离**，不应标 ⚠️偏乐观。
> **判据**：⚠️/🔵/❓ 只看"分数与当前实测值的差"；"变好/变坏的方向"另放**趋势备注列**，两者分开，不得混用同一个标记。

### 6.3 背离处理规则（强制）

1. **偏乐观**：必须在报告中写明上调的**前瞻理由**（如"营收尚未兑现但在手订单/产能/渗透率拐点支撑"），并注明"打分高于当前财务表现"。**给不出前瞻理由就下修到数据区间**。
2. **偏保守**：检查是否存在被忽视的压制因素（周期顶部、一次性收益）；若无，提示这是潜在低估机会。
3. **数据缺失**：在校验列标 ❓，并在标的/结论处说明"基于定性判断，缺财务佐证"。
4. **趋势/周期备注（放独立列，不改校验状态）**：把"方向信号"单列——如毛利率同比上行/下滑、景气顶部的高毛利（周期性、不可持续）、一次性收益等。这些提示动量与可持续性，但 **不改变校验状态标记**（✅/⚠️/🔵/❓ 只看"分数 vs 当前实测值"，见 6.2）。例：毛利率 34% 打利润 3 分＝✅一致，其"同比 -5.6pct"写进本备注列，不标 ⚠️。

### 6.4 输出要求

报告"高价值方向识别"部分需在三高热力矩阵之后，附一张**财务校验表**，逐环节列出：
增长锚 / 利润锚 / 壁垒锚 的实测值、数据来源与时间，**外加两个分开的列**：
1. **校验状态**（✅/⚠️/🔵/❓）——只标"分数 vs 数据"的对齐度（见 6.2）。
2. **趋势/周期备注**——毛利上行/下滑、周期顶部、一次性收益等方向信号（见 6.3.4），**与校验状态严格分列，不混用**。
若第一步裸分被审计改动，在环节名或分数处保留 `4→3` 留痕（见 6.0）。
所有取自联网的数字标注来源机构与时间；预测值与历史实际值要区分标注。

## 七、估值与赔率（好行业 ≠ 好价格）

三高解决"是不是好生意"，本节解决"现在是不是好价格"。**前面判断再好，撞上高估值也可能没收益**。每个核心标的/环节都要过这一关。

### 7.1 选对估值工具
| 公司类型 | 主用估值 | 说明 |
|---|---|---|
| 成长型（高增速） | PE / **PEG** / PS | 利润尚薄时用 PS；PEG 看成长是否被透支 |
| 重资产 / 制造 | PB / EV-EBITDA | 结合 ROE 看 PB 是否合理 |
| 周期型 | PB + 周期位置 | 切忌用景气顶部低 PE 判断便宜（PE 陷阱） |
| 平台 / 软件 | PS / EV-Sales | 未盈利期看收入倍数 + 留存 |

### 7.2 三个必算指标（联网取数）
1. **PEG** = 当前 PE ÷ 未来 2–3 年净利 CAGR。`<1` 偏低估、`1–1.5` 合理、`>2` 偏贵。
2. **历史估值分位**：当前 PE/PB 在过去 3–5 年的百分位（0–100%）。**>80% 分位=估值透支**，<30%=相对便宜。
3. **隐含预期（预期差）**：当前价格已 price in 多少增速？把它与三高判断的增速比——
   - 市场预期 < 你的判断 → **正预期差**（机会）
   - 市场预期 > 你的判断 → **负预期差**（透支，易杀估值）

#### 7.2.1 优先用脚本取一手分位（治本，推荐）
历史分位不必"靠猜"。若运行环境能跑 Python，**优先尝试调用 `assets/fetch_valuation.py`** 取真实日频序列算出的分位：

```bash
python assets/fetch_valuation.py 688017 002050 601689 300124 --md
```

- 输出：每只标的的 **PE(TTM)/PB/PS 当前值 + 近 N 年历史分位**（真实序列计算，可直接引用，标注"来源：akshare/东财 + 取数日期"）。
- **PEG**：脚本会输出东财口径 PEG，但净利下滑/微利时自动打 `⚠️失真` 标记——**打标者不可盲信，仍降级为定性**。

**依赖缺失时的处理流程（硬规则，防止擅自改用户环境）：**
1. 直接跑脚本；若报 `缺少 akshare/pandas`（ModuleNotFoundError），**不要自动 `pip install`**——装包是改动用户环境的动作。
2. 一句话告知用户可选安装命令（`pip install -r assets/requirements.txt`），并**征得同意后**再装；用户不想装就跳过。
3. 未安装时，**直接回退到 7.2.2 防编造约束**，估值分位降级为定性（历史高位/中位/低位），报告照常产出——脚本是可选增强，非必需。
4. 已装但取数失败（网络/接口变动/停牌）时同样回退定性，**不猜数**。

#### 7.2.2 防编造硬约束（脚本取不到时必守）
PEG 依赖"未来净利 CAGR 预测"、历史分位依赖"过去 3–5 年 PE/PB 时间序列"——WebSearch 常拿不全，是本 skill 幻觉风险最高的地方。**脚本能取就用脚本；脚本也取不到时**：
> - 拿不到可靠来源（脚本失败、无券商一致预期、无行情终端分位）时，**一律标 ❓ 数据缺失，并降级为定性区间**（如"估值处于历史高位/中位/低位""PEG 偏高/合理"），**严禁凭空捏造具体数值**。
> - 给出的每个估值数字必须能指到来源与时间（脚本取的标"akshare/东财+日期"；用预测净利算 PEG 时注明"预测年份 + 来源机构"）。编不出来源就不写数字。
> - 报告中宁可留白标"待用户用一手数据补全"，也不要给一个看似精确、实则编造的分位/PEG。

### 7.3 赔率框架
- **上行空间**：乐观情景目标市值 ÷ 当前。**下行空间**：悲观/估值底 ÷ 当前。
- **赔率 = 上行 ÷ 下行**。结合胜率（=三高+校验给出的确定性）分类：
  - 高胜率 + 低赔率 → 确定性底仓（贵但稳）
  - 低胜率 + 高赔率 → 弹性博弈（便宜但需催化兑现）
  - 高胜率 + 高赔率 → 最优（罕见，常出现在出清期/预期低点）
  - 低胜率 + 低赔率 → 回避

### 7.4 价值 × 估值 四象限（核心结论可视化）
将每个高价值环节/标的落到四象限：
- **高价值 · 低估值** → 重点机会（报告高亮）
- **高价值 · 高估值** → 好公司贵价格，等回调 / 已透支，标注"赔率不足"
- **低价值 · 低估值** → 价值陷阱嫌疑，查是否结构性衰退
- **低价值 · 高估值** → 回避

### 7.5 输出要求
报告"投资机会综合"部分需附：
1. **价值×估值四象限图**（标的散点）。
2. **估值分位表**：标的 | 当前 PE/PB | 历史分位 | PEG | 隐含预期 vs 判断（正/负预期差）| 赔率结论。
数字均标来源与时间；估值用预测净利时注明预测年份与来源。**不给目标价、不给买卖建议**，只呈现估值位置与赔率。

## 八、空头视角 / Bear Case（反共识对冲，每份报告必做）

联网检索天然放大**共识叙事**——热门行业的正面信息更多、更响，模型顺着写就会变成"卖方研报口径"。前面所有环节（三高、估值、主线）都在论证"为什么值得看"；本节强制切换立场，专门论证"为什么可能是错的"，用以对冲乐观偏差。

### 8.1 为什么单独设一节
"证伪条件"若只附在主线卡片后面，容易写成敷衍的一句话。Bear Case 要求**认真扮演空头**，独立成卡，与多头论据等重。一份没有强空头论据的报告是不完整的。

### 8.2 每份报告必产出的 Bear Case 卡片
针对 1–3 条投资主线，逐条回答：
1. **最强的三条空头论据**：产能过剩/价格战、技术路线被颠覆、政策/补贴退坡、需求证伪、估值已完全 price in、竞争格局恶化、海外龙头降维打击等——挑该主线最要命的三条，具体到可观察变量。
2. **这条主线最可能错在哪一环**：驱动假设里哪个环节最脆弱（是需求、是格局、还是价格）。
3. **一句话证伪信号**：出现什么**具体、可量化**的数据就应判定主线证伪并退出（如"某季度行业库存周转 > X 个月""龙头毛利率连续两季 < Y%""渗透率停在 Z% 不再上行"）。

### 8.3 与"共识温度"的对照（可选加分）
若能判断，标注当前市场对该主线的**共识温度**（冷门/升温/拥挤）：
- 拥挤共识 + 高估值 → 空头论据权重加大，提示"反身性/踩踏"风险。
- 冷门 + 低估值 → 多头论据可信度相对更高（预期差空间大）。

> 输出要求：报告"投资机会综合"部分，在主线卡片之后附独立的 **Bear Case 卡片**（可用与多头卡片对照的样式），至少覆盖上述 1、3 两点。

## 九、价值迁移判断（给三高快照补上时间维度）

三高矩阵是产业**当下**的价值快照，但产业链的价值分布是**动态**的。投资的超额收益往往来自"卡住价值即将迁入的环节"，而不是"价值当前最高的环节"——后者常已被市场 price in。本节强制对每个环节判断价值的**流向**。

### 9.1 判断每个环节的迁移方向
对三高矩阵里的每个环节，标 **↑迁入 / ↓迁出 / →稳定**，并给一句驱动理由。常见迁移驱动：

| 驱动 | 价值往哪走 | 例 |
|---|---|---|
| **一体化 vs 解耦** | 一体化 → 价值向整合方集中；解耦 → 价值分散回专业环节 | 锂电/光伏一体化挤压单一环节利润 |
| **技术路线变迁** | 价值从旧路线迁向新路线的卡位环节 | 长时储能/新电芯路线；先进制程 |
| **国产替代推进** | 替代完成后，利润从"稀缺卡脖子环节"再分配到规模制造/下游 | 卡脖子环节国产化率从低到高后溢价收敛 |
| **上下游议价权转移** | 供需/集中度变化改变谁拿走利润 | 上游资源景气回落，利润回流中游龙头 |
| **渗透率阶段切换** | S 曲线陡升段价值在放量制造；成熟后迁向品牌/服务/运营 | 微笑曲线右移 |

### 9.2 与三高、估值的联动规则（核心）
把"当前价值高低"与"迁移方向"叠加成四种组合：

- **当前高价值 + ↑迁入 → 最优象限**（报告重点高亮）：好生意且还在变更好。
- **当前高价值 + ↓迁出 → 警惕"已见顶"**：往往已被 price in，容易撞上**负预期差**（呼应第七节）——高分不等于好机会。
- **当前中低价值 + ↑迁入 → 潜在预期差机会**：市场还没给分，是正预期差的高发地，值得下探。
- **当前低价值 + ↓迁出 → 回避**：结构性衰退。

### 9.3 输出要求
1. 三高热力矩阵**增加一列"价值迁移 ↑/↓/→"**，每格附一句驱动理由。
2. 高价值象限的判定叠加迁移方向，明确标出"高价值且迁入"的最优象限。
3. 迁移判断多为前瞻定性，**证据不足时说明依据（趋势/在建产能/政策方向），不硬编数字**；重大迁移假设应在 Bear Case 里给出对应的证伪信号。
